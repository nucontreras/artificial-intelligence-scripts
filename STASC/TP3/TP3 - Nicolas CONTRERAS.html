Illustrer l'instabilit des arbres CART sur des arbres de petites tailles (prendre par exemple max_leaf_nodes=6). On pourra tirer des chantillons alatoirement avec remise dans les donnes initiales (prcdure bootstrap) en utilisant la fonction resample et comparer finalement (graphiquement) les arbres de dcision ainsi ajusts.

In [17]:
from sklearn.utils import resample
import matplotlib.pyplot as plt

fig, ax = plt.subplots()

n_echantillons = np.arange(1, 101, 1)
mln_list = np.arange(2, 6, 1)

for mln in mln_list:
    predictions = []
    for i in n_echantillons:
        bootstrap_size = int(0.8 * len(X_Spam))
        X_Spam_resampled, Y_Spam_resampled = resample(X_Spam, Y_Spam, n_samples=bootstrap_size, random_state=i, replace=True) 
        Base_DecisionTree = DecisionTreeClassifier(max_leaf_nodes=mln)
        Base_DecisionTree.fit(X_Spam_resampled, Y_Spam_resampled)
        score_ = Base_DecisionTree.score(X_Spam_resampled, Y_Spam_resampled)
        predictions.append(score_)
    ax.plot(n_echantillons, predictions, label=f'max_leaf_nodes={mln}, mean={round(mean(predictions),3)}')

ax.grid()
ax.set_xlabel("Echantillon i")
ax.set_ylabel("Score")
ax.set_title("Instabilit des arbres CART sur des arbres de petites tailles")
ax.legend(loc="upper center", bbox_to_anchor=(0.5, 1.3), ncol=2, shadow=True);

Avec sckit-learn, les performances en rgression sont values par dfaut via le score du $R^2$ (coefficient d'ajustement ou coefficient de dtermination). Ce score donne une mesure de la qualit de la prdiction :

$$ R^{2}=1- \frac {\sum _{i=1}^{n}(y_{i}-{\hat {y_{i}}})^{2}}{\sum _{i=1}^{n}(y_{i}-{\bar {y}})^{2}} = 1- \frac { \frac 1n \sum _{i=1}^{n}(y_{i}-{\hat {y_{i}}})^{2}}{ var (Y)}. $$

Un ajustement parfait correspond videmment $R^2 = 1$.

On reconnait un terme d'erreur quadratique au numrateur. On peut bien sr calculer le score sur l'chantillon d'apprentissage ou sur l'chantillon test.

Expliquer pourquoi il est naturel que le score OOB soit plus faible lorsque le nombre d'arbres est faible.¶

Car oob utilise le nombre d'arbres. Comme il y a trop peu d'arbres au dbut, les rsultats ne sont pas bons.

Lorsque la fort contient peu d'arbres, on note que certaines observations ne sont dans auncun des chantillons OOB, d'o les warnings.

Lorsque le nombre d'arbres est suffisamment grand, le score OOB finit par dpasser le score valu par validation croise. Ce phnomne peut s'expliquer par le fait que dans le cas OOB, nous utilisons toutes les donnes pour construire la fort alatoire, alors que dans le cas de la validation croise, les forts alatoires ne sont construites que sur les 4/5 ime des donnes, il est donc naturel que les performances soient suprieures dans le premier cas.

Pour ce qui concerne maintenant les performances de la mthode OOB, en tant que mthode d'estimation de l'erreur de gnralisation, la littrature rcente (voir par exemple les conclusions de cette tude) indique que cette mthode tend surestimer l'erreur de gnralisation. Il s'agit d'un sujet de recherche encore ouvert.

On note dans le modle prcdent que pour centaine d'arbres, le score de la fort alatoire tend ce stabiliser un plateau. On va donc garder n_estimators= 100dans la suite, mme si dans l'idal il faudrait aussi faire varier ce paramtre dans ce qui suit.
